# 프로젝트 요구사항 명세서 (Requirements Specification)

## 프로젝트 개요

**프로젝트명**: Ideal Type World Cup & Quiz Game Platform  
**목적**: 사용자가 직접 이상형 월드컵과 퀴즈를 생성하고 공유할 수 있는 인터랙티브 게임 플랫폼 구축  
**대상 사용자**: 모든 연령대의 웹 사용자

---

## 1. 기능적 요구사항 (Functional Requirements)

### 1.1 월드컵 도메인 (Worldcup Domain)

#### 1.1.1 월드컵 목록 조회
- **요구사항**: 사용자는 생성된 월드컵 목록을 조회할 수 있어야 한다.
- **상세 스펙**:
  - 페이징 지원 (`_limit`, `_page` 파라미터)
  - 정렬 기능 (조회수, 플레이 횟수, 생성일 기준)
  - 카테고리별 필터링
- **API**: `GET /worldcups?_sort=viewCount&_order=desc&_limit=9`

#### 1.1.2 월드컵 게임 플레이
- **요구사항**: 사용자는 토너먼트 방식으로 게임을 플레이할 수 있어야 한다.
- **게임 규칙**:
  - 라운드 선택: 8강, 16강, 32강
  - 매 라운드마다 2개의 후보 중 1개 선택
  - 라운드 진행: 32강 → 16강 → 8강 → 4강 → 결승
  - 우승자 결정 및 TOP 4 기록

#### 1.1.3 월드컵 생성
- **요구사항**: 로그인한 사용자는 새 월드컵을 생성할 수 있어야 한다.
- **필수 입력**: 제목, 설명, 카테고리, 썸네일, 후보 최소 8개
- **제약 조건**: 인증된 사용자만 접근 가능

---

### 1.2 퀴즈 도메인 (Quiz Domain)

#### 1.2.1 퀴즈 게임 플레이
- **요구사항**: 제한 시간 내에 문제를 풀어야 한다.
- **게임 규칙**:
  - 각 문제당 제한 시간 (기본 10초)
  - 정답 입력 후 즉시 다음 문제로 이동
  - 시간 초과 시 오답 처리
- **점수 계산**: 정답 10점 + (남은 시간 × 2점)

#### 1.2.2 티어 시스템
- **티어 기준**:
  - 멘사: 180점 이상
  - 수재: 160~179점
  - 우등생: 120~159점
  - 모범생: 80~119점
  - 평범: 30~79점
  - 노력필요: 30점 미만

---

### 1.3 인증 시스템 (Authentication)

#### 1.3.1 회원가입
- **필수 입력**: 이메일, 비밀번호, 닉네임
- **유효성 검사**: 이메일 형식, 비밀번호 최소 6자, 닉네임 2~10자

#### 1.3.2 로그인
- **인증 방식**: JWT (JSON Web Token)
- **토큰 저장**: localStorage
- **자동 로그인**: 페이지 새로고침 시 토큰 확인

#### 1.3.3 인증 가드 (Protected Routes)
- **보호 페이지**: `/mypage`, `/worldcup/create`, `/quiz/create`
- **미인증 접근 시**: 로그인 페이지로 리다이렉트

---

## 2. 비기능적 요구사항 (Non-Functional Requirements)

### 2.1 성능
- 페이지 로딩 시간: 초기 로딩 3초 이내
- API 응답 시간: 평균 500ms 이내

### 2.2 사용성
- 반응형 디자인: 모바일, 태블릿, 데스크톱 지원
- 직관적인 UI/UX

### 2.3 보안
- XSS, SQL Injection 방지
- JWT 토큰 만료 처리
- CORS 설정

### 2.4 유지보수성
- 코드 주석 완비
- 일관된 네이밍 규칙
- 문서화 (README, API 명세, 요구사항)

---

## 3. 기술 스택

- **Frontend**: Vue.js 3.5, Pinia, Element Plus, Vite
- **Backend**: JSON Server (Mock API)
- **HTTP Client**: Axios
- **브라우저**: Chrome, Firefox, Safari, Edge (최신 버전)
